# README For Part 3 of my Website

This is a student project for **Cashbuild Limited**, a leading building materials retailer in South Africa.

**The website was created to showcase what Cashbuild offers and provide information to customers. It includes:**

Home page – introduction and overview of Cashbuild

About Us – company history, mission, and vision

Products & Services – details and images of building materials and tools

News Page – updates, promotions, and community projects

Contact Us – contact form, store information, and location map

The project was coded in plain HTML (with a logo and basic navigation) to demonstrate how a simple, structured website works.

**Purpose**

To simulate a real company website

To practice HTML page structure, navigation, and content organization

To provide a simple, user-friendly interface for visitors to learn about Cashbuild

**Features Implemented**

Consistent navigation bar across all pages

Hero/cover sections with headings and subheadings

Product and service listings with images styled via CSS

News & Promotions page with announcements and grid layout

Contact form with validation for name, email, and message

Embedded Google Maps showing store location

Customer reviews section with JavaScript + localStorage to keep reviews even after page reload

Navigation & Layout

Consistent navigation bar across all pages

Hero/cover sections with headings and subheadings

Grid and flexbox layouts for responsive design

Cream white background for readability with black headers and text

Product & Service Display

Products organized into categories: Kitchen, Bathroom, Bedroom, Living Room

Interactive buttons to select a category and display products dynamically

Products appear as cards with placeholder images, product names, and prices

Grid layout ensures responsive display across devices

JavaScript used to dynamically generate product cards from data objects

**Interactivity**

Contact form validation prevents empty submissions

Customer reviews submitted by users are displayed dynamically

Reviews are saved using localStorage to remain visible after page reload

Interactive category buttons update product listings dynamically without refreshing the page

**Technical Details**

HTML – structured multi-page layout with semantic elements

CSS – consistent styling, including colors, fonts, spacing, and responsive grid/flexbox layouts

JavaScript – dynamic rendering of products and reviews, form validation, and localStorage integration

Example Product Display (Template)

Each item displayed as a card with a placeholder image

Same template applies to Bathroom, Bedroom, and Living Room categories
